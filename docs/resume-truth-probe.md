# Resume Truth Probe

`简历拷打` 模式的目标不是让用户背项目稿，而是模拟真实技术一面里对项目真实性的连续追问。

## Probe Chain

项目追问按下面顺序推进：

1. 背景和瓶颈：项目解决什么问题，为什么值得做。
2. 个人贡献：哪些设计、编码、联调、上线和复盘动作是本人完成的。
3. 技术选型：比较过哪些替代方案，为什么当前方案更合适。
4. 指标证明：指标来源、统计口径、时间窗口、上线前基线、上线后对比和归因方式。
5. 可验证证据：PR、日志、压测报告、监控看板或告警记录能否证明结论。
6. 故障复盘：现象、影响范围、根因、恢复动作、兜底和长期改进。

## Risk Flags

简历解析会标记项目风险点：

- 缺少量化指标或指标口径。
- 个人贡献边界不清。
- 技术选型缺少替代方案和取舍。
- 没有故障、风险或复盘证据。
- 存在“高并发、显著优化、大幅提升”等模糊表述。

这些风险点会进入后续追问和报告改写建议。

## Answer Standard

高分项目回答应该能被验证：

- 不把团队成果包装成个人成果。
- 不只说“用了 Redis/MySQL/RAG”，而是说明为什么这么选。
- 不只说“提升很多”，而是说明指标口径和基线。
- 不回避故障和边界，能讲清恢复和长期改进。
- 能指出可以验证自己贡献的代码、日志、监控或压测证据。
