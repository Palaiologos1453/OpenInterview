# Campus CS Interview Design

OpenInterview 当前聚焦 Java 后端校招，默认面试流程应贴近真实技术面。

## 标准流程

1. 自我介绍。
2. 简历项目深挖。
3. 计算机基础。
4. 岗位专项。
5. 系统设计入门或场景题。
6. 反问。
7. 报告和复习计划。

## 方向

- Java 后端：Java 基础、JVM、Spring、OS、网络、数据库、缓存、并发、分布式基础、系统设计。

前端、测开、数据、算法、SRE、嵌入式等方向暂不开放到目录。原因不是不能做，而是当前项目要优先保证 Java 后端题库、报告和复练链路真实可用，避免多方向入口稀释质量。

## 难度

- 实习面试：重基础，项目可以是课程项目。
- 普通校招：基础、项目、算法均衡。
- 大厂校招：重追问、边界条件、复杂度和工程细节。
- 头部冲刺：高压追问、开放问题和系统设计入门。

## 评分维度

- 计算机基础。
- 项目深度。
- 场景分析。
- 表达沟通。
- 岗位匹配。

每个维度都应该输出证据和改进建议，避免只有总分。

## 复练资料

报告除了总分和逐题反馈，还应输出：

- 复练题。
- 推荐回答结构 / 示例答案。
- 参考答案。
- 常见错误。
- 面试官追问点。
- 低分回答 vs 高分回答。
- 关联知识点。

这些内容优先来自题库 metadata、rubric、followups 和候选人的实际缺口，不能只生成泛泛模板。

## 简历拷打

简历模式应先做轻量结构化分析：

- 自动拆项目卡片。
- 抽个人贡献。
- 找虚假、模糊或难验证表述。
- 追问指标来源和统计口径。
- 追问技术选型依据和替代方案。
- 追问故障、风险兜底和复盘。
